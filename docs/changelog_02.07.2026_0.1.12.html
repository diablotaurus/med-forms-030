Версия 0.1.12 — кроссплатформенное развёртывание

Приложение теперь можно штатно запускать не только на Windows Server, но и на двух независимых Linux-серверах в разных зданиях. Каждый экземпляр использует собственную локальную базу, настройки и пользователей.

  • Поддержаны Ubuntu Server 24.04 LTS и Debian 12.
  • Добавлена служба systemd с автоматическим запуском.
  • Реализованы установка, запуск, обновление и резервное копирование.
  • Зафиксированы безопасные права и запрет запуска приложения от root.

Этапы разработки (0.1.11 → 0.1.12)

0.1.11Поддержка Linux-серверов

  • ДОБАВЛЕНО Создан отдельный комплект Bash-сценариев в scripts/linux/ для Ubuntu Server 24.04 LTS и Debian 12.
  • setup.sh создаёт виртуальное окружение Python, устанавливает зависимости и готовит рабочие каталоги.
  • start.sh выполняет ручной production-запуск через Waitress.
  • install-service.sh создаёт и запускает службу systemd; приложение работает от обычного пользователя, запуск от root заблокирован.
  • update.sh останавливает службу, создаёт резервную копию SQLite, получает код с GitHub, обновляет зависимости и запускает службу.
  • backup.sh создаёт датированные копии base.db и поддерживает ротацию по количеству сохранённых копий.
  • uninstall-service.sh удаляет только службу, сохраняя код, настройки и базу данных.
  • Добавлен .gitattributes: Bash-скрипты сохраняются с LF и исполняемыми правами в Git.
  • Проверены Bash-синтаксис, исполняемые права и реальное создание бэкапа.
