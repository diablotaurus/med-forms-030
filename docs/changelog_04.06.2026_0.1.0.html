История разработки — med-forms-030

Веб-приложение для учётных форм N 030/у-Д/с и N 030-ПО/у (карты диспансеризации и профилактического осмотра несовершеннолетнего)
Релиз 0.1.0 04.06.2026 Лицензия GPL v3.0 Python · Flask · SQLite

О документе

Это конспект этапов разработки приложения. Каждый шаг помечен минорной версией от 0.0.1 до 0.0.21 — в порядке появления. Версия 0.1.0 — первый сведённый релиз, объединяющий все шаги ниже.

Обозначения: ДОБАВЛЕНО ИСПРАВЛЕНО ДОКУМЕНТАЦИЯ

🏷 Версия 0.1.0 — сводный релиз

Полнофункциональное веб-приложение для заполнения, хранения и печати двух медицинских учётных форм. Ключевые возможности на момент релиза:

  • Две формы (030/у-Д/с и 030-ПО/у) с переключением через вкладки.
  • Динамические блоки диагнозов (1–10) с авто-нумерацией пунктов.
  • Маски и двойная (браузер + сервер) проверка дат и СНИЛС.
  • Авторизация, смена пароля, настройки реквизитов организации.
  • Аккуратная печать/PDF под A4; печать в одной вкладке.
  • Раздельная нумерация списков и корзина с восстановлением.
  • Открытый код под GNU GPL v3.0, опубликован на GitHub.

Этапы разработки (0.0.1 → 0.0.21)

0.0.1Базовое веб-приложение (форма 030/у-Д/с)

  • ДОБАВЛЕНО Flask + SQLite, файл базы base.db.
  • Полное воспроизведение формы N 030/у-Д/с: разделы 1–19, шапка, подписи, М.П.
  • Динамические блоки диагнозов в разделах 16 и 17 (минимум 1, до 10, кнопка «+ Добавить диагноз») с автоматическим пересчётом номеров пунктов и зависимых разделов.
  • Большие авто-растущие текстовые поля в пунктах 17.11, 17.12, 18, 19.
  • Список карт, создание / редактирование / удаление, печать под A4 через браузер (с «Сохранить как PDF»).
  • Все поля сохраняются в JSON-столбце data.
app.py · templates/* · static/css/style.css · static/js/form.js

0.0.2Маски ввода и проверка данных (в браузере)

  • ДОБАВЛЕНО Авто-маска дат дд.мм.гггг для всех 8 полей-дат.
  • Маска СНИЛС ХХХ-ХХХ-ХХХ ХХ.
  • Проверка корректности: календарные даты (месяц, день, високосные годы) и контрольная сумма СНИЛС.
  • Подсветка ошибочных полей, сводка ошибок сверху, блокировка сохранения.

0.0.3Серверная (дублирующая) валидация

  • ДОБАВЛЕНО Проверка дат и СНИЛС на сервере (validate_form).
  • При некорректных данных карта не сохраняется, форма возвращается со списком ошибок (HTTP 422).
  • Защита от обхода браузерной проверки (прямые запросы, отключённый JS).

0.0.4Авторизация

  • ДОБАВЛЕНО Страница входа, сессии в cookie, постоянный secret.key.
  • Таблица users, пароли хранятся хэшами (werkzeug).
  • Защита всех страниц декоратором login_required.
  • Учётная запись по умолчанию admin / admin, кнопка «Выйти».

0.0.5Смена пароля из интерфейса

  • ДОБАВЛЕНО Страница смены пароля с проверками (текущий пароль, длина нового, совпадение, отличие от старого).
  • Доступ через кнопку и по клику на имя пользователя.

0.0.6Упрощение доступа к смене пароля

  • ИСПРАВЛЕНО Убрана отдельная кнопка «Сменить пароль».
  • Смена пароля теперь только по клику на имя пользователя — единообразно на всех страницах.

0.0.7Вторая форма 030-ПО/у и мультиформенность

  • ДОБАВЛЕНО Карта профилактического медицинского осмотра (разделы 1–18 + подписи).
  • Динамические диагнозы в разделах 15 и 16 второй формы.
  • Столбец form_type в БД + автоматическая миграция; маршруты /forms/<тип>.
  • Меню-вкладки переключения форм; списки фильтруются по типу.
