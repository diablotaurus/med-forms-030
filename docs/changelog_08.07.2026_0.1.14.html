Версия 0.1.14 — автосохранение и совместная работа

Пользователь больше не теряет введённые данные, если отвлёкся или не нажал кнопку «Сохранить». Несколько сотрудников могут одновременно работать с разными картами, а конфликт одной карты обнаруживается до перезаписи.

  • Серверное автосохранение обеих форм и понятный индикатор состояния.
  • Черновики новых карт с последующим окончательным сохранением.
  • Оптимистическая блокировка по ревизии и HTTP 409 при конфликте.
  • SQLite WAL и ожидание блокировки для 3–5 одновременных пользователей.
  • Согласованные резервные копии SQLite/WAL.

Этапы разработки (0.1.13 → 0.1.14)

0.1.13Автосохранение и защита совместного редактирования

  • ДОБАВЛЕНО Изменения автоматически отправляются на сервер через 2 секунды после окончания ввода.
  • Панель формы показывает состояния «Есть несохранённые изменения», «Сохранение…», время успешного сохранения и ошибки связи.
  • Первая правка новой карты создаёт запись с отметкой «Черновик»; обычная кнопка «Сохранить» выполняет полную валидацию и завершает карту.
  • Медицинские данные не помещаются в localStorage браузера.
  • НАДЁЖНОСТЬ В таблицу карт добавлены revision, is_draft и updated_by.
  • Устаревшая вкладка не может молча перезаписать правки другого пользователя: сервер возвращает HTTP 409 и требует перезагрузить карту.
  • В списке показываются черновики и пользователь последнего изменения.
  • SQLite переведена в режим WAL; установлен busy_timeout=10000 мс.
  • Бэкапы Windows и Linux используют SQLite Backup API, включая последние WAL-транзакции, и проходят PRAGMA quick_check.
  • Добавлены автоматические тесты: завершение черновика, конфликт ревизий и одновременная работа пяти независимых сессий.
