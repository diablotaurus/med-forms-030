# Участие в проекте

Спасибо за интерес к проекту! Любой вклад приветствуется — исправления ошибок,
улучшения, документация.

## Как сообщить об ошибке или предложить идею

Откройте **Issue** на GitHub и опишите:

- что вы делали и что ожидали увидеть;
- что произошло на самом деле (текст ошибки, скриншот);
- какая форма (030/у-Д/с или 030-ПО/у), браузер и ОС.

## Как внести изменения (Pull Request)

1. Сделайте форк репозитория и создайте ветку от `main`:

   ```bash
   git checkout -b fix/краткое-описание
   ```

2. Поднимите окружение и запустите приложение:

   ```bash
   python -m venv .venv
   # Windows:  .\.venv\Scripts\Activate.ps1
   # Linux/macOS:  source .venv/bin/activate
   pip install -r requirements.txt
   python app.py
   ```

3. Внесите изменения, проверьте, что приложение запускается и обе формы
   (создание, сохранение, печать) работают.
4. Сделайте коммит с понятным сообщением и откройте Pull Request в `main`,
   описав суть изменений.

## Стиль кода

- **Python** — стиль PEP 8, отступы 4 пробела, осмысленные имена.
- **JavaScript** — отступы 4 пробела, `"use strict"`, без сторонних
  зависимостей (ванильный JS).
- **HTML/CSS** — придерживайтесь существующей структуры шаблонов и классов;
  стили экрана и печати лежат в `static/css/style.css`.
- Комментарии и текст интерфейса — на русском языке, как в остальном проекте.
- Не коммитьте локальные данные и секреты: `base.db`, `secret.key`, `.venv`
  (они уже в `.gitignore`).
